# ТЗ для вайбкодинг-агента — «Атлас физиогномики» (учебное приложение)

## Что строим
Интерактивное веб-приложение для самостоятельного изучения физиогномики как **языка восприятия лиц**. Не диагностика людей — учебный атлас + тренажёр.

## Данные (уже готовы в папке)
- `data.json` — структура: zones, features[].variants[] (id, label, image, tradition, perception), principles[].
- `images/` — 14 PNG-портретов (вымышленные лица), имена совпадают с полем `image` в data.json.
- `АНАЛИЗ_примеров.md` — развёрнутый разбор каждого фото (можно показывать как «подробнее»).
- Файлы `01_*.md … 10_*.md` — теория по каждой черте.

**Контракт данных:** приложение читает только `data.json` и `images/`. Ничего не хардкодить — всё из JSON, чтобы атлас легко дополнялся новыми фото.

## Экраны (MVP)
1. **Атлас** — сетка карточек по черте. Карточка: фото + label + короткий `perception`. Фильтр по `feature`.
2. **Деталь черты** — крупное фото, блоки «Традиция» (`tradition`) и «Как читается» (`perception`), плюс контраст-пара (например круг ↔ овал).
3. **Тренажёр (флешкарты)** — показываем фото, пользователь угадывает черту/трактовку, переворот карточки = ответ. Счёт.
4. **Принципы** — экран с `principles[]` и общим дисклеймером.

## Обязательное требование — дисклеймер
На главном экране и в футере вывести поле `disclaimer` из data.json: физиогномика не наука, `tradition` = фольклор, `perception` = восприятие. Это не косметика — это защита проекта от скатывания в стереотипы.

## Стек (на усмотрение, рекомендация)
- Лёгкий фронт: **React + Vite + Tailwind**, без бэкенда, данные из локального `data.json`.
- Мобайл-фёрст, тёмная/светлая тема.
- Деплой статикой (Vercel/Netlify).

## Чего НЕ делать
- Не добавлять загрузку фото реальных людей и «анализ по фото пользователя» — это юридически и этически грязно.
- Не выдавать вердиктов о характере. Формулировки только «так читается / так трактует традиция».

## Возможное развитие (v2)
- Дополнить атлас недостающими контрастами (узкие глаза, курносый нос, тонкие губы, треугольное лицо).
- Режим «собери образ»: миксуешь черты → приложение описывает, как такой образ считают.
- Квиз с прогрессом и spaced-repetition.
